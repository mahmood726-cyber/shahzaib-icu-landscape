# Data Availability Statement

## Primary Data Sources

- **ClinicalTrials.gov**: All trial registration data is publicly available via the
  [ClinicalTrials.gov API v2](https://clinicaltrials.gov/data-api/about-api).
  No authentication required. Data is in the public domain (US government work).

- **PubMed**: Article metadata retrieved via
  [NCBI E-utilities](https://www.ncbi.nlm.nih.gov/books/NBK25501/).
  Free for programmatic access with rate limits (3 requests/second without API key).

## Enrichment Sources

The enrichment pipeline queries the following open APIs (no authentication required
unless noted):

| Source | Access | Authentication |
|--------|--------|----------------|
| PubMed efetch | Free | None (polite-pool email recommended) |
| OpenAlex | Free | None |
| Crossref | Free | None (polite-pool email recommended) |
| Europe PMC | Free | None |
| OpenCitations | Free | None |
| Unpaywall | Free | Email required |
| OpenFDA/FAERS | Free | None |

## Derived Data

All derived datasets (CSV, Parquet, JSON summaries) are generated by the pipeline
from the above public sources. The complete pipeline code is provided to enable
full reproduction.

## Data Dictionary

See `build_living_map.py` for field definitions. Key output columns:

- `nct_id`: ClinicalTrials.gov identifier
- `brief_title`: Study title
- `overall_status`: Registration status (RECRUITING, COMPLETED, etc.)
- `keyword`: Matched hemodynamic keyword
- `normalized_keyword`: Canonical category (35 categories)
- `comparator_type`: placebo / sham / usual_care / active_control / none
- `data_sources`: Origin registry (ctgov, pubmed, or both)

## Reproducibility

To regenerate all data from scratch:

```bash
pip install -r requirements.txt
python living_update.py --force-full
```

The pipeline fetches fresh data from public APIs. Results may vary slightly
between runs due to registry updates and PubMed indexing delays.
